Command Reference

Easy-to-use commands for maintaining your Discord server

Dust Manager Commands

All commands use Discord's slash command system for intuitive, predictable operation

Mass Unban Commands

/unbanall
Unban all users from the server. This will remove all bans from your server's ban list.

Member Purge Commands

/purge @role @excluded-role
Kick members with a specific role. You can optionally exclude members with another role from being kicked.

Channel Cleanup Commands

/cleanup-enable #channel
Restrict channel to admin-only messages and bot commands
/cleanup-disable #channel
Remove admin-only restriction from a channel
/cleanup-list
Show all channels with cleanup mode enabled

Message Clearing Commands

/clear-channel Time #channel
Delete specified number of messages in a channel based on time period
/clear-user @user Time #channel
Delete messages from a specific user within a time period in a channel

Access Control Commands

/access-grant @role Category
Grant command access to a role for a specific category of commands
/access-revoke @role Category
Revoke command access from a role for a specific category
/access-list
List all command access permissions currently configured

Usage Guide

Learn how to effectively use Dust Manager's powerful cleanup tools

Mass Unban Guide

The mass unban feature allows you to unban multiple users at once, giving them a second chance to join your server.

  • Make sure Dust Manager has the appropriate permissions to manage bans in your server.
  • Use the command to unban all users:
    /unbanall
  • You'll receive a confirmation message showing how many users will be unbanned. Confirm to proceed:
    Ready to unban 45 users. Type "confirm" to proceed or "cancel" to abort.
  • Once complete, you'll receive a log of all unbanned users for your records.
  • This is especially useful after resolving a raid situation or when you want to give banned users a fresh start.

Member Purge Guide

The member purge utility allows you to remove users with specific roles from your server.

  • Ensure Dust Manager has the "Kick Members" permission in your server.
  • To purge members with a specific role:
    /purge @role @excluded-role
  • You'll receive a confirmation message with the affected user count:
    This will remove 23 members from the server. Type "confirm" to proceed or "cancel" to abort.
  • After confirmation, members will be kicked and you'll receive a log of the operation.
  • This feature is useful for cleaning up inactive members or removing users with temporary roles after an event.

Channel Cleanup Guide

Convert regular channels into command-only channels to reduce clutter while still allowing bot commands.

  • Ensure Dust Manager has "Manage Channels" and "Manage Messages" permissions.
  • To set up a command-only channel:
    /cleanup-enable #channel
  • Now the channel will automatically delete any messages that don't appear to be bot commands.
  • To disable command-only mode:
    /cleanup-disable #channel
  • To see which channels have cleanup mode enabled:
    /cleanup-list

Message Clearing Guide

Quickly remove multiple messages based on various criteria to keep your channels clean.

  • Ensure Dust Manager has the "Manage Messages" permission.
  • To clear messages in a channel:
    /clear-channel Time #channel
  • To clear messages from a specific user:
    /clear-user @user Time #channel
  • After execution, you'll receive a confirmation of how many messages were deleted.
  • This feature is perfect for cleaning up after events, removing spam, or maintaining a tidy server.

Best Practices

Important considerations when using powerful cleanup tools

Use With Caution

Dust Manager provides powerful tools that can make significant changes to your server. Please follow these best practices:

  • Always double-check the confirmation messages before proceeding with mass actions.
  • Restrict access to Dust Manager commands to trusted administrators only using the access control commands.
  • Consider creating backups or logs before performing large-scale operations like mass unbans or member purges.
  • Start with smaller test operations before performing server-wide changes.
  • Mass unbans should be used carefully, especially if your server has experienced targeted raids.
  • Remember that purged members will need a new invite to rejoin the server.
  • Message clearing operations are permanent and cannot be undone.
  • Set up proper role hierarchies to ensure only authorized personnel can use destructive commands.
