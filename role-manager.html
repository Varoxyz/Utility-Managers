Streamline your Discord server's role system with these powerful features

Auto Roles on Join

Automatically assign roles to new members when they join your server. Set up multiple auto roles to instantly give new members the permissions they need.

Perfect for welcome roles, verification tiers, or basic permission sets that all new members should receive upon joining your community.

Role Persistence

Make roles permanent for your members, even if they leave and rejoin. Role Manager remembers which roles users had and automatically reassigns them when they return to your server.

This prevents role loss during temporary absences and provides consistent user experiences across your Discord community.

Command Reference

Easy-to-use commands for managing roles in your Discord server

Role Manager Commands

All commands use Discord's slash command system for intuitive control

Auto Role Commands

/autorole-add @role
Add an auto-assigned role to new members
/autorole-remove @role
Remove a role from auto assignment
/autorole-list
List all auto roles
/autorole-delay @role minutes:10
Set a delay before role is assigned to new members

Role Persistence Commands

/rolepersist-add @role
Make roles permanent for users
/rolepersist-remove @role
Remove protected role status
/rolepersist-list
Show roles that persist when members rejoin
/rolepersist-status @user
Check which persistent roles a user has

Role Combiner Commands

/rolecombiner-add @triggerRole @targetRole action:"add"
Create a rule to add targetRole when triggerRole is assigned
/rolecombiner-add @triggerRole @targetRole action:"remove"
Create a rule to remove targetRole when triggerRole is assigned
/rolecombiner-remove @triggerRole @targetRole
Remove a role combination rule
/rolecombiner-list
List all role combination rules

Role Sync Commands

/rolesync-add @localRole serverId:"123456789" remoteRoleId:"987654321"
Sync roles between servers
/rolesync-remove @localRole serverId:"123456789"
Remove role sync
/rolesync-list
List all role syncs
/rolesync-status
View sync status and last sync time

Blocked Roles Commands

/blockrole-add @role
Block a role from being assigned
/blockrole-remove @role
Remove a role from blocked list
/blockrole-list
List blocked roles
/blockrole-exception @role @allowedRole
Add exception for a blocked role

Mass Roles Commands

/add-role @roleToAdd
Add a role to all members
/add-role @roleToAdd @filterRole
Add a role to members with filterRole
/remove-role @roleToRemove
Remove a role from multiple members
/swap-roles @oldRole @newRole
Replace one role with another
/role-cancel
Cancel ongoing role operations

Access Control Commands

/access-grant @role command:"command-name"
Grant command access to a role
/access-revoke @role command:"command-name"
Revoke command access
/access-list
List command access permissions
/access-reset
Reset access settings to default

Usage Examples

See how Role Manager can help organize your Discord server

Setting Up Verification Tiers

  • Create a "New Member" role with limited permissions
  • Set it as an auto role with /autorole-add @New Member
  • Create a "Verified" role with standard permissions
  • When members verify, give them the Verified role
  • Use /rolecombiner-add @Verified @New Member action:"remove" to automatically remove New Member role when Verified is assigned
View Auto Role Commands View Role Combiner

Cross-Server Role Synchronization

  • Add Role Manager to both your main and secondary servers
  • Create matching roles in each server (e.g., "Supporter")
  • Use /rolesync-add @Supporter serverId:"123456789" remoteRoleId:"987654321" to link the roles between servers
  • Members who receive the role in one server will automatically get it in the other
  • Perfect for multi-server communities or partner networks
View Role Sync Commands

Role Management During Events

  • Create an "Event Participant" role for a special server event
  • During event signup, assign this role to interested members
  • Use /rolepersist-add @Event Participant to make it persistent
  • After the event, use /remove-role @Event Participant
  • For returning events, previous participants will automatically get their roles back
